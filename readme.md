# Техническое задание: Агрегатор кредитов "CreditMatch"

## 1. Формальное ТЗ (в стиле SRS)

### 1.1. Назначение системы
Система позволяет пользователю пройти идентификацию, заполнить кредитную анкету, сравнить предложения банков и отследить статус обработки заявки в реальном времени.

### 1.2. Функциональные модули

#### Модуль 1: Авторизация (Login)
- **Поле ввода:** Номер телефона (маска: `+7 (___) ___-__-__`).
- **Механика:** Отправка 4-значного SMS-кода.
- **UI элементы:**
    - Кнопка "Получить код" (активна после ввода 10 цифр).
    - Таймер повторной отправки (60 сек).
    - OTP-поле (4 отдельных инпута с автофокусом).

#### Модуль 2: Анкета заемщика
- **Поля ввода:**
    - **Сумма кредита:** Слайдер (диапазон 50 000 – 5 000 000 руб., шаг 10 000 руб.).
    - **Срок кредита:** Сегментированный контрол (6, 12, 24, 36 мес.).
    - **Тип занятости:** Выпадающий список (Наёмный сотрудник / Самозанятый / ИП).
    - **Ежемесячный доход:** Поле ввода с маской цифр.
- **Валидация:** Кнопка "Подобрать банки" неактивна, пока все поля не заполнены корректно.

#### Модуль 3: Список банков
- **Отображение:** Карточки банков (скролл).
- **Содержимое карточки:**
    - Логотип, название банка.
    - Процентная ставка (жирным, крупно).
    - Ежемесячный платёж (мелко).
    - Кнопка "Выбрать".
- **Сортировка:** По умолчанию — по возрастанию процентной ставки.

#### Модуль 4: Экран ожидания решения
- **Триггер:** Нажатие на кнопку "Выбрать" в карточке банка.
- **Отображение процесса (пошагово):**
    1.  `Проверка персональных данных...`
    2.  `Запрос в кредитное бюро...`
    3.  `Формирование финального предложения...`
- **Визуализация:** Спиннер + текст смены статусов.
- **Результат:** Успех (сумма + ставка) / Отказ (причина).

### 1.3. Критерии приемки
- Существует сквозной сценарий (Вход -> Список банков -> Анкета -> Профиль).
- Из анкеты есть возврат к списку банков.
- Из профиля можно перейти к списку банков, а из списка банков — в профиль.
- Все состояния кнопок (disabled, hover, loading) стилизованы.
- Сайт реализует требования WCAG

## 2. Неформальное ТЗ (для дизайнера / презентации)

**Проект:** *Кредитный навигатор*  

### Экран 1. Вход
Напоминаем пользователю, что мы его помним.
**Как это выглядит:**
- Заголовок: *"Ваш личный консультант"*.
- Поле "Телефон".
- Под ним четыре квадратика для кода из СМС.
- **Фишка:** Надпись *"Не переживайте, мы не звоним"* (чтобы снять страх спама).

### Экран 2. Анкета (4 вопроса)
Долгие анкеты пугают. Оставляем только суть.

| Вопрос | Что используем | Почему |
| :--- | :--- | :--- |
| **Сколько денег?** | Слайдер (Тяни ползунок) | Визуально сразу видно рост платежа. |
| **На сколько лет?** | 3 большие кнопки (1 год, 2 года, 3 года) | Удобно для пальца на телефоне. |
| **Кто вы?** | Дропдаун с иконкой 🧑‍💼 | Пользователи любят иконки. |
| **Ваш доход?** | Просто цифры (без копеек) | Снижаем тревожность. |

### Экран 3. Витрина банков
Пользователь должен понять, куда выгодно бежать, за 3 секунды.
**Принцип верстки:**
1.  **Топ-1** (Самая низкая ставка) — выделяем зеленым цветом.
2.  Остальные — серыми карточками.
3.  В каждой карточке глаз цепляется только на **проценты** (огромный шрифт) и **кнопку**.

### Экран 4. Ожидание
Экран ожидания обобрения заявки.
**Текст под спиннером должен меняться:**
> *Ищем ваш паспорт в базе...*  
> *Спрашиваем у робота Сбера...*  
> *Почти готово! Загружаем одобрение...*

**Итоговый дизайн (2 варианта):**
- *Успех:* Зеленый градиент. "Одобрено 500 000 ₽ под 18%".
- *Отказ:* Светло-серый. "Попробуйте уменьшить сумму или выберите Почта Банк".
